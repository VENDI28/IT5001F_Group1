# TODO: Apply Label Encoding to categorical columns

# Create copies to avoid modifying original data
X_train_encoded = X_train.copy()
X_test_encoded = X_test.copy()

# Dictionary to store label encoders for each column
label_encoders = {}

for col in categorical_cols:
  le = LabelEncoder()
  #
  #     # Fit on training data
  le.fit(X_train_encoded[col].astype(str))
  #
  #     # Transform train and test
  X_train_encoded[col] = le.transform(X_train_encoded[col].astype(str))
  #
  #     # Handle unseen categories in test set

  X_test_map_col = X_test_encoded[col].astype(str).fillna("__MISSING__")
  class_to_int = {cls: idx for idx, cls in enumerate(le.classes_)}
  X_test_mapped = X_test_map_col.map(class_to_int)
  X_test_encoded[col] = X_test_mapped.fillna(-1).astype(int)

  #
  #     # Store encoder
  label_encoders[col] = le

# TODO: Compute evaluation metrics for all three models

def evaluate_model(y_true, y_pred, model_name):
    accuracy = accuracy_score(y_true, y_pred)
    precision = precision_score(y_true, y_pred, average='weighted')
    recall = recall_score(y_true, y_pred, average='weighted')
    f1 = f1_score(y_true, y_pred, average='weighted')

    return {
        'Model': model_name,
        'Accuracy': accuracy,
        'Precision': precision,
        'Recall': recall,
        'F1-Score': f1
    }

Selected Model: Random Forest

Justification based on:

  1. Evaluation Metrics:

    • It achieved the best Accuracy, Precision, Recall, and F1-score among the three models

  2. Class Imbalance Considerations:

    • From the confusion matrices, Logistic Regression largely collapses to the majority class (class “0”), with very few true positives in minority classes. The Random Forest still favors the majority, but it recovers meaningful true positives for minority classes

  3. Model Complexity and Interpretability:

    • Logistic Regression: Low complexity, high interpretability via coefficients/odds ratios, that can directly indicate the direction and strength of each feature’s effect on the log‑odds of a class

    • Decision Tree: Low complexity, single tree is highly interpretable (can be visualized) and explained as rule-based splits (“if–then” decisions).

    • Random Forest: Moderate model complexity. Far less prone to overfitting than a single tree due to averaging across many trees. Interpretability is medium as we can still inspect feature importances, permutation importances, and partial dependence/SHAP to explain predictions.

  4. Practical Deployment Considerations:

    • Although not as transparent as Logistic Regression or a shallow tree, Random Forest can still be explained using well‑established tools (feature importance, permutation importance, SHAP). This helps with troubleshooting and stakeholder communication.
    • Better ease of maintenance. Unlike a single decision tree, Random Forest does not require delicate tuning to prevent overfitting. Retraining the model periodically with new data is straightforward, and the model generally maintains consistent performance over time.
    • Lastly, it is efficient at inference time. Despite being an ensemble of many trees, Random Forest prediction is fast because each tree makes a simple, rule‑based decision. The ensemble averages these decisions efficiently, making it suitable for real‑time or near‑real‑time applications.

Why this model is preferred over others:

  • Versus Logistic Regression: Logistic Regression is simple and interpretable, but its linear nature limits predictive power in datasets with complex, non‑linear relationships. It also collapses heavily towards predicting the majority class under imbalance. Random Forest captures richer patterns and achieves notably higher F1‑scores without requiring complex transformations.

  • Versus Decision Tree: A single Decision Tree is interpretable, but unstable and prone to overfitting—small fluctuations in the data can produce vastly different trees. This instability is reflected in its lower scores. Random Forest overcomes these weaknesses through ensembling, producing smoother, more reliable predictions and better generalization.

Logistic Regression:

  • High interpretability. Coefficients (after encoding and scaling) directly indicate the direction and strength of each feature’s effect on the log‑odds of a class. You can report odds ratios, confidence intervals, and global feature importance from absolute coefficient magnitudes.
